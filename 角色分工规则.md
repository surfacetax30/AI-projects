# 项目角色分工规则

> 适用项目：OTS Approval Helping Agent 及后续所有项目  
> 最后更新：2026-05-26

---

## Rule 1: 角色边界

**Trae Solo（产品设计会话）只做产品设计。工程实现全部由 Trae Solo Code 完成。**

| 角色 | 负责内容 | 红线 |
|------|---------|:--:|
| **Trae Solo** | PRD · 需求分析 · 业务流程 · Agent 架构 · 状态机 · 置信度/SFT/纠错 · .drawio 图 · .docx 文档 · 数据表设计 · 基础设施配置 · 设计评审 · 准备清单 · 规格转让书 | **禁止写业务代码** |
| **Trae Solo Code** | Agent 业务逻辑 · API 路由 · 数据库迁移 · 测试用例 · 部署脚本 · 应用启动 | **接收规格文档后编码** |

### 产品设计会话可以做：
- 生成 .drawio 架构图、.docx PRD 文档、.md 规划文档
- 讨论和评审架构设计
- 提供数据库表结构
- 配置 .env / docker-compose.yml / requirements.txt
- 在 work/ 目录下写临时处理脚本

### 产品设计会话不能做：
- 写 Agent 业务逻辑代码
- 写 API 路由和业务接口
- 写数据库迁移脚本（Alembic）
- 写测试用例
- 写完整的应用启动逻辑

## Rule 2: 对话记忆机制

### 触发条件
当对话 token 消耗接近 **0.8M tokens**（约 80% 上下文窗上限）时，主动执行记忆流程。

### 执行流程

1. **判断**：每次回复前估算 token 消耗，若接近 0.8M，暂停当前任务
2. **生成记忆**：在项目文件夹生成 `对话记忆_[日期].md`，包含：
   - 所有已确认的设计决策（不可推翻清单）
   - 当前进度（已完成 / 进行中 / 待处理）
   - 项目文件索引
   - 下一个会话的启动指令
3. **通知用户**：告知记忆文件已生成，新会话读取即可接续
4. **新会话恢复**：按优先级读取 `对话记忆.md` → PRD → 架构图 → 准备清单

## 开发流程

```
Trae Solo（本会话）
    │
    ├── 理解需求 → PRD → 架构图 → 设计决策
    │
    ├── 产出规格转让书.md（含所有设计规格 + 待实现清单）
    │
    └── 对话接近 0.8M → 生成对话记忆.md
            │
            ▼
Trae Solo Code（新会话）
    │
    ├── 读 对话记忆.md → 重建上下文
    ├── 读 规格转让书.md → 了解待实现功能
    ├── 读 PRD.docx + .drawio → 理解产品全貌
    │
    └── 按 Sprint 计划开始编码
```
