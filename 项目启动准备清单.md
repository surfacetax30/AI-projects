## 一、开发环境搭建

### 1.1 基础环境

- [ ] Python 3.10+ 确认安装：`python3 --version`
- [ ] 创建虚拟环境：`python3 -m venv venv && source venv/bin/activate`
- [ ] 升级 pip：`pip install --upgrade pip`

### 1.2 核心依赖

```
fastapi          # Web 框架
uvicorn          # ASGI 服务器
sqlalchemy       # ORM
asyncpg          # PostgreSQL 异步驱动
alembic          # 数据库迁移
pydantic         # 数据校验
python-multipart # 文件上传
```

- [ ] `pip install fastapi uvicorn sqlalchemy asyncpg alembic pydantic python-multipart`

### 1.3 文件与邮件

```
minio            # 对象存储客户端
imapclient       # IMAP 邮件监听
python-docx      # Word 报告生成（可选，MVP 后期）
```

- [ ] `pip install minio imapclient`
- [ ] MinIO 部署：`docker run -d -p 9000:9000 -p 9001:9001 --name minio minio/minio server /data --console-address ":9001"`

### 1.4 数据库

- [ ] PostgreSQL 安装（本地或 Docker）：
  ```bash
  docker run -d --name ots-pg -e POSTGRES_PASSWORD=ots2026 -e POSTGRES_DB=ots -p 5432:5432 postgres:16
  ```
- [ ] 创建数据库：`ots`
- [ ] 初始化 Alembic：`alembic init migrations`

### 1.5 LLM API

- [ ] DeepSeek API Key 就绪（已有 DeepSeek-v4 Pro）
- [ ] 在项目中创建 `.env` 文件：
  ```
  DEEPSEEK_API_KEY=sk-xxx
  DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
  ```
- [ ] `.env` 加入 `.gitignore`

## 三、模拟数据生成计划

### 3.1 需要生成的测试数据

| 数据项 | 数量 | 用途 |
|--------|------|------|
| 模拟 OTS 邮件 | 10 封 | 测试邮件网关过滤和分类 |
| 模拟测试报告 PDF | 5 份 | 测试解析 Agent 字段提取 |
| 模拟供应商声明（扫描件风格） | 3 份 | 测试模糊文档解析 |
| 零件主数据 | 5 个零件 | 测试完整性检查 |
| 异常案例（含 FAIL 项） | 3 份报告 | 测试异常识别 |

### 3.2 邮件模板内容

每封模拟邮件需包含：
- 主题：`[测试报告] P2024-XXXX DV测试报告`
- 发件人：`lab_emc@supplier.com`
- 附件：PDF 报告文件
- 正文：简短的测试说明

### 3.3 PDF 报告模板内容

每份模拟报告需包含：
- 零件号、零件名称
- 测试类型（EMC/DV/HIL/软件/CAN/静态电流/诊断）
- 测试日期
- 测试结果列表（每项标记 PASS/FAIL/BORDERLINE）
- 实验室名称
- 正常案例 ×4，异常案例（含 FAIL）×1

## 六、开发约束 & 原则

1. **单进程优先**：所有 Agent 在同一个 FastAPI 进程中运行，拆分是代码模块而非微服务
2. **Mock 先行**：LLM 调用加超时 + 重试 + fallback，不在核心流程中硬依赖 API
3. **测试友好**：每个 Agent 的输入输出明确，可独立测试，不依赖真实邮箱
4. **PE 视角驱动**：每写一个功能就自问"PE 看到这个会怎么做"，保持用户思维
5. **先跑通再优化**：Sprint 0-1 目标是"一封模拟邮件到一条数据库记录"，不做 Prompt 调优、不做置信度校准
