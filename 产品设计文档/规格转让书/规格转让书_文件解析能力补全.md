## 〇、问题描述

### 当前数据流（有缺口）

```
POST /api/tasks/{id}/reports  上传 PDF/DOCX/XLSX
    ↓
  文件字节流 → MinIO 存储 ✅
    ↓
  发布 report.received 事件 ✅
    ↓ payload = {filename, storage_path, file_size}
    ↓  ❌ 缺少 report_text 字段
    ↓
Parser.process() 读取 payload.report_text
    ↓ 取不到 → 降级为 "[Mock content for xxx.pdf]"
    ↓ 解析的是占位符，不是真实报告内容
```

### 缺口根因

- `app/api/tasks.py` 的 `upload_report()` 发布事件时不带 `report_text`
- `POST /api/webhooks/mail` 同样不带 `report_text`
- Parser 只消费纯文本，不消费二进制文件

---

## 一、实现目标

上传任意格式文件后，Parser 自动拿到该文件的实际文本内容，无需人工介入。

### 支持的文件格式

| 格式 | 扩展名 | 提取方式 | 新增依赖 |
|------|--------|---------|---------|
| PDF | `.pdf` | `pypdf` 提取文本 | `pypdf` |
| Word | `.docx` | `python-docx` 提取段落文本 | `python-docx` |
| Excel | `.xlsx` `.xls` | `openpyxl` 读取 → 转 markdown 表格 | 已有 `openpyxl` |
| CSV / TSV | `.csv` `.tsv` | 直接 decode 为文本 | 无 |
| 纯文本 | `.txt` `.md` | 直接 decode | 无 |
| 图片 | `.png` `.jpg` `.jpeg` | base64 编码 + 多模态 System Prompt（V2） | — |
| 其他 | `.zip` 等 | 不提取，记录 "unsupported" | — |

**MVP 范围：优先实现 PDF / DOCX / XLSX / CSV / TXT。图片多模态留到 V2。**

## 二、代码产物

### 2.1 新建文件：`app/services/extractor.py`（~80 行）

```python
"""文件文本提取器 — 将 PDF/DOCX/XLSX/CSV/TXT 转为纯文本，供 Parser 消费。"""

import io, csv, logging
from pathlib import Path

logger = logging.getLogger(__name__)

# 最大提取字符数（与 Parser 的 4000 字截断对齐）
MAX_CHARS = 4000


def extract_text(filename: str, file_bytes: bytes) -> str:
    """主入口：根据扩展名分发到对应提取器，返回纯文本字符串。"""
    ext = Path(filename).suffix.lower()

    handlers = {
        ".pdf":  _extract_pdf,
        ".docx": _extract_docx,
        ".xlsx": _extract_xlsx,
        ".xls":  _extract_xlsx,      # openpyxl 也支持 .xls
        ".csv":  _extract_csv,
        ".tsv":  _extract_csv,       # CSV 方言处理
        ".txt":  _extract_txt,
        ".md":   _extract_txt,
        ".png":  _extract_image,
        ".jpg":  _extract_image,
        ".jpeg": _extract_image,
    }

    handler = handlers.get(ext)
    if handler is None:
        return f"[不支持的文件格式: {ext}]"

    try:
        text = handler(file_bytes)
        return text[:MAX_CHARS]
    except Exception as e:
        logger.warning(f"文本提取失败 ({filename}): {e}")
        return f"[文本提取失败: {e}]"


# ── 各格式提取器 ──

def _extract_pdf(data: bytes) -> str:
    """pypdf — 逐页提取文本。"""
    from pypdf import PdfReader
    reader = PdfReader(io.BytesIO(data))
    pages = []
    for page in reader.pages:
        t = page.extract_text()
        if t:
            pages.append(t.strip())
    return "\n\n".join(pages) if pages else "[PDF 无可提取文本]"


def _extract_docx(data: bytes) -> str:
    """python-docx — 逐段落提取。"""
    from docx import Document
    doc = Document(io.BytesIO(data))
    paragraphs = []
    for para in doc.paragraphs:
        text = para.text.strip()
        if text:
            paragraphs.append(text)

    # 也提取表格内容
    for table in doc.tables:
        for row in table.rows:
            cells = [cell.text.strip() for cell in row.cells]
            paragraphs.append(" | ".join(cells))

    return "\n".join(paragraphs) if paragraphs else "[DOCX 无可提取文本]"


def _extract_xlsx(data: bytes) -> str:
    """openpyxl — 逐 Sheet 提取，转 markdown 表格。"""
    from openpyxl import load_workbook
    wb = load_workbook(io.BytesIO(data), data_only=True)
    sheets = []
    for name in wb.sheetnames:
        ws = wb[name]
        rows = []
        for row in ws.iter_rows(values_only=True):
            cells = [str(c) if c is not None else "" for c in row]
            if any(cells):  # 跳过全空行
                rows.append(" | " + " | ".join(cells) + " |")
        if rows:
            sheets.append(f"## Sheet: {name}\n" + "\n".join(rows))
    return "\n\n".join(sheets) if sheets else "[XLSX 无可提取数据]"


def _extract_csv(data: bytes) -> str:
    """CSV/TSV — csv 模块解析。"""
    text = data.decode("utf-8", errors="replace")
    delimiter = "\t" if "\t" in text[:200] else ","

    reader = csv.reader(io.StringIO(text), delimiter=delimiter)
    rows = []
    for row in reader:
        if any(row):
            rows.append(" | " + " | ".join(str(c).strip() for c in row) + " |")
    return "\n".join(rows) if rows else "[CSV 无可提取数据]"


def _extract_txt(data: bytes) -> str:
    """纯文本 — 直接 decode。"""
    return data.decode("utf-8", errors="replace")


def _extract_image(data: bytes) -> str:
    """图片 — MVP 不做多模态，返回占位说明。"""
    return "[图片文件 — MVP 阶段不支持图片解析，请升级到 V2 多模态版本]"
```

## 六、边界与约束

1. **硬截断**：`extract_text()` 统一在 4000 字符处截断，与 `parser.py` 的 `text_content[:4000]` 对齐（Parser 本身也截断，双重保险）
2. **异常安全**：任何格式的提取失败不会中断上传——事件仍然正常发布，只是 `report_text` 带 `[错误信息]` 前缀
3. **不修改事件 Schema**：只在 payload dict 中新增 `report_text` key，不影响现有 consumer
4. **图片暂不支持**：当前 DeepSeek V4 (`deepseek-reasoner`) 对图片多模态支持有限，V2 升级后再做
